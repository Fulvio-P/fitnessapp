1.  Andare nella cartella front-end e dare il comando npm run build.
    Questo genera la cartella dist, che contiene la nostra app buildata.
2.  Andare nel server express.
    Nelle prime righe, dove mettiamo le varie app.use(...), aggiungere:
    app.use(express.static("../front-end/dist"))
    Questo segnala a express che la cartella che gli abbiamo passato contiene i file html.
    Attenzione: il path relativo dipende dalla pwd da cui eseguiremo node,
        non dalla posizione del file.
3.  Andare nella cartella back-end e lanciare il server express da lì (node server/index.js).
    (La pwd da usare dipende dal path relativo dato al punto precedente.
     In questo caso, è back-end.)
4.  Andare sul browser e digitare localhost:5000/ oppure localhost:5000/index.html
    (va da sè che la porta effettiva è quella su cui si è aperto il server express)
    (pare che se non si specifica un file da richiedere il server di default ne cerchi uno
     chiamato specificamente "index.html")
5.  Enjoy ;)

N.B.: Questo è il modo in cui funziona.
      Ho provato anche ad aprire il Live Server di VSC e andare in frontend/dist,
      ma non ha mostrato nulla. Il browser dava errori di non riuscire a caricare
      i file JS linkati.

P.S.: Per accedere direttamente a punti diversi dell'app,
      prendiamo ad esempio la pagina di about.

      Inserire direttamente localhost:5000/#/about funziona.

      Ma quel cancelletto è un po' di troppo, no?

      Possiamo farne a meno impostando un endpoint del server express
      che ha il solo scopo di fare un redirect, usando l'apposita funzione res.redirect
      Questa accetta URL assolute o anche path locali a partire dalla root del server,
      quindi basta redirectare verso "/#/about" ed ecco che il browser si ritroverà
      a visualizzare localhost:5000/#/about, che è proprio l'url della pagina di about
      che funziona se inserita direttamente come dicevo prima.

      Il caricamento diretto della pagina di about è stato verificato con successo
      con una sessione in incognito appena aperta. (path /about)
